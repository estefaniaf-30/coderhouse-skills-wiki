🔌 Plugins instalados
Los plugins son paquetes que extienden las capacidades de Claude. Cada plugin agrupa varias skills relacionadas. Estos son los que tenemos activos en Coderhouse.
📖 ¿Cómo empezar con las Skills?
Guías paso a paso para usar, instalar y crear tus propias skills en Cowork.
🚀 Cómo activar una skill
1
Escribile a Claude de forma natural
No hace falta ningún comando especial. Claude detecta automáticamente qué skill usar según lo que escribís. Por ejemplo, si escribís "qué cancelaciones hubo esta semana", se activa la skill de cancelaciones sola.
💡 Tip: Usá frases del día a día. Claude entiende contexto, no necesitás tecnicismos.
2
O usá el comando directo con "/"
Algunas skills tienen un comando corto. Escribiendo ese comando la activás directamente, sin importar cómo lo formulés.
/cancelaciones /revision-diaria /retencion-cx
3
Claude va a pedirte los datos que necesita
Muchas skills hacen preguntas de clarificación (ej: "¿qué período querés consultar?"). Respondé y Claude arranca con la tarea.
🔌 Cómo instalar una skill o plugin desde Cowork
1
Abrí el panel de Customize
En la barra lateral de Cowork, hacé clic en el ícono de Customize (o el engranaje ⚙️). Esto abre el panel de configuración de tu sesión.
2
Buscá la skill o plugin
En el panel vas a ver una sección de Skills y otra de Plugins. Usá el buscador para encontrar lo que necesitás por nombre o por área.
💡 Tip: Podés buscar por nombre técnico (ej: intercom-metricas) o por área (ej: CX, pagos).
3
Activala con un clic
Hacé clic en la skill o plugin que querés activar. Si es un plugin, puede pedirte conectar una cuenta (Intercom, Slack, Google Drive, etc.) — Claude te guía en ese proceso.
4
Listo, ya podés usarla
Una vez activada, la skill aparece disponible en tu sesión. Solo escribile a Claude de forma natural y la va a detectar automáticamente.
💡 Tip: Si no sabés si ya tenés la skill activa, simplemente preguntale a Claude: "¿Qué skills tengo disponibles?"
🔌 Cómo instalar un plugin pidiéndoselo a Claude
1
Pedile a Claude que busque el plugin
Describí lo que necesitás y Claude va a buscar en el registro de plugins disponibles.
Ejemplo: "Quiero conectar Notion a mi Cowork" Ejemplo: "Hay algún plugin para Jira?"
2
Confirmá la instalación
Claude te va a mostrar una vista previa del plugin y sus skills. Confirmás y queda instalado en tu sesión de Cowork.
3
Conectá las credenciales si hace falta
Algunos plugins (como Intercom, Google Drive, Slack) requieren autenticación. Claude te guía paso a paso para conectar tu cuenta.
💡 Tip: Una vez conectado, el plugin queda disponible en todas tus sesiones de Cowork.
🛠️ Cómo crear una skill propia
1
Activá la skill "Skill Creator"
Esta skill meta te ayuda a construir nuevas skills. Describile a Claude qué tarea repetitiva querés automatizar.
Ejemplo: "Quiero crear una skill que genere un reporte diario de NPS" Ejemplo: "Ayudame a crear una skill para buscar datos en nuestra BD"
2
Claude arma el archivo SKILL.md
La skill queda definida en un archivo de texto con: nombre, descripción, frases de activación e instrucciones paso a paso para Claude.
3
Testeala y refinala
Probá la skill con distintas frases. Si Claude no la activa bien, pedile que optimice la descripción. La skill creator también puede correr evaluaciones automáticas.
💡 Tip: Cuanto más específica sea la descripción de la skill, mejor la detecta Claude automáticamente.
4
Compartila con el equipo
Una vez que la skill funciona bien, podés empaquetar el archivo en un plugin para que todo el equipo la tenga disponible. Usá la skill "Crear Plugin" para hacer el paquete y distribuirlo.
Decile a Claude: "Quiero empaquetar esta skill en un plugin para compartirla"
📚 Glosario
Conceptos clave para entender cómo funciona el ecosistema de Claude + Cowork.
🧠 La analogía más fácil
📱
Claude = el teléfono
La base inteligente que entiende todo y puede hacer muchas cosas.
🏪
Plugin = una app
Extiende las capacidades de Claude para un área específica. Lo "instalás" una vez.
⚡
Skill = una función de esa app
Una tarea específica que Claude sabe hacer gracias al plugin. Se activa por contexto.
🖥️
Cowork = el sistema operativo
El entorno donde todo convive: Claude + plugins + skills + tus herramientas conectadas.
Skill
habilidad
Una instrucción detallada que le dice a Claude cómo completar una tarea específica. Es un archivo de texto (SKILL.md) con pasos, contexto y ejemplos.
Ejemplo "intercom-metricas" le enseña a Claude cómo consultar la API de Intercom, qué equipos existen, cómo calcular el TMR y cómo presentar los resultados.
Plugin
extensión
Un paquete que agrupa varias skills relacionadas, conectores a herramientas externas (MCPs) y configuraciones. Se instala una vez y queda disponible para toda la sesión.
Ejemplo El plugin "coderhouse-api" agrupa todas las skills de gestión: estudiantes, cohorts, pagos, catálogo, certificados.
MCP
Model Context Protocol
El protocolo estándar que permite a Claude conectarse con herramientas externas (Slack, Google Drive, Intercom, etc.) y ejecutar acciones reales en ellas.
Ejemplo El MCP de Intercom permite que Claude busque conversaciones, lea métricas y analice tickets directamente desde tu cuenta.
Cowork
modo colaborativo
El entorno de escritorio de Claude que combina acceso a archivos, shell de Linux, plugins y skills para tareas operativas complejas. Está pensado para equipos no técnicos.
Ejemplo Desde Cowork podés pedirle a Claude que genere un reporte de cancelaciones, lo guarde en Drive y lo mande por Slack, todo en un solo mensaje.
Artifact
artefacto
Un archivo HTML interactivo que Claude genera y que persiste entre sesiones. Se abre en el sidebar de Cowork y puede actualizarse con datos frescos.
Ejemplo Este mismo diccionario es un artifact: Claude lo creó, vive en tu Cowork y puede actualizarse cuando se suman nuevas skills.
Schedule
tarea programada
Una tarea que Claude ejecuta automáticamente en intervalos de tiempo (diario, semanal, etc.) sin que vos tengas que pedirlo.
Ejemplo La revisión diaria de cohorts puede estar programada para correrse todos los días a las 8am y mandar el resumen automáticamente por Slack.
